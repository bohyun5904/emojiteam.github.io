[참고]
-- (StyleGAN, DCGAN, PGGAN의 개념) --
StyleGAN: 이미지 생성에 대한 질감, 품질을 높이는거에 초점으로 세부적인 이미지를 생성할때 Good
DCGAN: GAN에서 가장 기본적인 형태로써 간단하게 빠르게 만들어짐
PGGAN: 작은 해상도의 이미지 생성 -> 점차적으로 해상도를 높여가며 높은 해상도의 이미지를 생성하여 정교하게
---------------------------------------------------------------------------------------------------------------------

-- (Facial expression, recognition의 개념) --
Facial expression(얼굴 인식) 기술: 얼굴의 (눈, 코, 입, 귀)등과 같은 구성 요소와 더불어 얼굴의 윤곽선, 주름 등과 
같은 더 깊은 특징을 식별, 추출하고 분류할 수 있다.

recognition(표정 인식) 기술: 얼굴 인식 기술과 연관된 기술
얼굴 이미지를 분석하여 해당 얼굴을 인식 -> 얼굴 주인의 정보를 확인하는 기술
감정 인식 분야에서는 얼굴 표정 분석을 통해 감정을 인식하고 분류가능함.
---------------------------------------------------------------------------------------------------------------------

⑤. DB설계
다음은 DB설계입니다. 같은내용의 뒤에 페이지에서 설명하겠습니다.
USER테이블에 사용자 id, 이름, 이메일, 패스워드 그리고 계정생성일, 최근접속일이 있고
EMOJIS에는 이모티콘 고유번호, 파일경로, 이름, 생성일
EMOJI_PERMISSIONS에는 권한 고유번호, 계정아이디, 이모티콘 고유번호
CHAT_ROOMS에는 사용자 ID, 초대한 사용자 아이디, 초대받은 사용자 아이디
CAHT_MESSAGES에는 사용자 ID, 메시지와 채팅방 고유번호, 메시지 타입과 내용, 이모티콘 고유번호가 있습니다.

⑥. 시스템 모듈 상세 설계
다음은 시스템 모듈 상세 설계입니다. 웹에서 이모티콘 생성 순으로 설명하겠습니다.

로그인과 회원가입은 POST방식으로 회원가입시에는 DB에 데이터 저장, 로그인시에는 입력받은 데이터를 DB와 비교하여 
로그인이 가능하며
사진 등록 및 삭제로는 하단의 예시코드처럼 이미지 파일 경로를 이용하여 등록하고
하단의 예시코드처럼 등록된 이미지와 경로를 서버로 전송합니다.

이모티콘 공유 게시판은 axios(엑시오스) 라이브러리를 이용하여 이모티콘공유 글 (등록, 삭제, 등록된 글)을 볼수있게 합니다.

다음으로는 이모티콘생성 상세설계입니다.

얼굴인식 모듈은 사용자가 업로드한 사진에서 사용자의 얼굴을 인식하여 인식여부에 따라 논리값을 반환합니다.
얼굴인식이 될 때 다음과정을 수행하도록 합니다.

사진 크기 및 해상도 조절 모듈은 GAN Target이미지로 사용할때를 위해서 얼굴을 검출하고 나서 이미지에서 
사용자의 얼굴 상대적인 위치를 기준으로 이미지를 확대시키고 해상도를 변경하고 이미지를 반환합니다.

얼굴 특징점 강조 모듈은 개인적 특성을 강조하기 위해 사용자의 얼굴에서 특징점을 추출하고 특징점간의 거리를 
계산하여 특출난 부분을 감지하고 특출난 부분을 강조시킨 다음에 이미지를 반환합니다.

이미지 생성 모듈은 GAN의 경쟁모델을 통해서 사용자 사진에서 새로운 이미지를 생성하고 새롭게 생성된 이미지를 반환합니다. 

밝기, 명암비 조절 모듈은 이미지 처리 기술을 이용하여 이미지의 밝기와 명암비를 조절한 뒤 이미지를 반환합니다.

스티커 추가 모듈은 스티커를 추가할 좌표를 명시하고 입력된 좌표에 스티커를 붙이고 이미지를 반환합니다.

폰트 추가 모듈은 내용과 글자 크기, 글꼴, 폰트를 추가할 좌표를 명시하고 이미지에 폰를 추가하고 반환합니다.

검색 알고리즘 모듈은 문자열을 받아서 해당 문자열과 가장 유사도가 높은 스티커 태그들을 배열형태로 반환합니다.
